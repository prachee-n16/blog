If we **understand** a program, we generally know:
- Inputs and Outputs: usually part of the function definition
- Static structure
	- data structures
	- Class Diagrams (UML)
	- design patterns
- Dynamic Structure
	- control structures/ call graph
	- pointers and aliasing
	- object diagrams 
	- Mutation
	- invariants
- Algorithms
	- runtime
		- asymptotic (Big-O)
		- constant factors
	- space usage 
		- asymptotic (Big-O)
		- constant factors
	- termination conditions
		- when will the program end?
